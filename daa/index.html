How Seam Carving Works

01

Compute Energy

Calculate the importance of each pixel using gradient-based energy — edges and details get high energy.

02

Build DP Table

Use Dynamic Programming to find the cumulative minimum energy path from top to bottom of the image.

03

Find Seam

Backtrack through the DP table to trace the minimum-energy vertical seam — the least noticeable path.

04

Remove & Repeat

Remove the seam (one column of pixels) and repeat to shrink the image while keeping important content.

Algorithm Deep Dive

🧩

Optimal Substructure

The minimum-energy seam ending at pixel (i, j) is composed of the minimum-energy seam ending at one of its three parents (i-1, j-1), (i-1, j), or (i-1, j+1) — plus the energy at (i, j) itself.

This means we can build the optimal solution from optimal sub-solutions, exactly what Dynamic Programming requires.

🔁

Overlapping Subproblems

Computing the minimum cost path to pixel (i, j) shares sub-computations with its neighbors. Without DP, the naïve recursive approach recalculates the same cells exponentially — O(3H).

DP stores each sub-result in a table, reducing this to O(W × H).

📐

Recurrence Relation

M[i, j] = Energy(i, j) + min( M[i-1, j-1], M[i-1, j], M[i-1, j+1] )

Where M[i, j] is the minimum cumulative energy of any seam starting from the top row and ending at pixel (i, j). The base case is M[0, j] = Energy(0, j) for all j.
